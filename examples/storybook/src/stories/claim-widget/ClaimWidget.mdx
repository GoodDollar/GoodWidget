  <DocsSection
    title="Why the shell is optional"
    description="MiniAppShell is layout chrome, not runtime plumbing. Use it when the widget should read as a section inside a larger app, or when you want to force a mobile-sized frame in demos and docs."
  >
    <DocsCallout title="Keep the runtime wrapper separate" tone="info">
      `ClaimWidget` already owns its internal `GoodWidgetProvider`. You only add `MiniAppShell` when
      you want a consistent frame around the widget, not when you want the widget to function.
    </DocsCallout>

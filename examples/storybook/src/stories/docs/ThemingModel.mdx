  <DocsSection

    <DocsGrid>
      <DocsCard title="Tokens">
        Raw design primitives like palette values, spacing, size scales, radii, and other values
        that should mean the same thing everywhere.
      </DocsCard>
      <DocsCard title="Themes">
        Semantic values like `background`, `color`, `borderColor`, and `shadowColor` that can vary
        by light or dark context.
      </DocsCard>
      <DocsCard title="Named component themes">
        Stable override targets like `light_Card` and `dark_Card` that let a host or widget author
        tune specific surfaces without rewriting the component.
      </DocsCard>
    </DocsGrid>
  </DocsSection>

    title="Decision rule"
    description="Choosing the right layer is more important than choosing a clever implementation."
  >
    <DocsList>
      <li>Change tokens when you want broad brand alignment across many surfaces.</li>
      <li>Change themes when you want semantic light or dark values to shift.</li>
      <li>Change named component sub-themes when you need a targeted surface adjustment.</li>

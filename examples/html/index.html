GoodWidget — Plain HTML Example

No React knowledge needed. Just a <script> tag and custom HTML elements.

Default (no overrides)

Just drop in the element — uses GoodDollar blue theme.

<gw-claim-widget></gw-claim-widget>

Pink brand (CSS custom properties)

Override colors from plain CSS — no JS needed.

.pink-widget {
  --gw-token-color-primary: #E91E63;
  --gw-token-color-primaryDark: #AD1457;
}

Teal brand (CSS custom properties)

Another brand override, same widget.

.teal-widget {
  --gw-token-color-primary: #00897B;
  --gw-token-color-primaryDark: #00695C;
}

JS overrides (widget component)

Override the widget's custom ClaimCard component + tokens via JavaScript.

el.themeOverrides = {
  tokens: { color: { primary: '#7B61FF' } },
  themes: {
    light_ClaimCard: { background: '#F3E5F5' }
  }
}

JS overrides (basic Card element)

Override the base Card theme — affects every Card-based component in the widget.

el.themeOverrides = {
  tokens: { color: { primary: '#FF6D00' } },
  themes: {
    light_Card: {
      background: '#FFF8E1',
      borderColor: '#FFB300'
    },
    light_Button: {
      background: '#FF6D00'
    }
  }
}

Wallet provider injection

Pass a wallet provider from your page so the widget can send transactions.

const el = document.getElementById('wallet-widget')
el.provider = window.ethereum

Iframe Embedding (Bridge)

Embed a third-party GoodWidget app in an iframe. The host's wallet is bridged via postMessage using @goodwidget/bridge. 

import { createIframeBridgeHost } from '@goodwidget/bridge/host'

const cleanup = createIframeBridgeHost({
  iframe: document.getElementById('widget-iframe'),
  provider: window.ethereum,
  allowedOrigins: ['https://widget.example.com'],
})

// In the embedded app's entrypoint:
import { enableIframeBridge } from '@goodwidget/bridge/child'
await enableIframeBridge({
  allowedParents: ['https://yourhost.com'],
  appId: 'claim-widget',
})
// window.ethereum is now a bridge provider
// Also discoverable via EIP-6963

The bridge supports EIP-6963 announcements so modern dapps can discover the provider without relying on window.ethereum.
